1. Covered Call — 备兑看涨期权

Mildly Bullish Income Low Complexity

Definition 策略定义

持有100股正股的同时，卖出1份虚值(OTM) Call期权。通过出售上涨权利换取期权金收入。适合温和看涨、愿意在一定价格卖出股票的投资者。

Payoff Structure 收益结构

Position: Long 100 shares at S₀ + Short 1 Call at strike K (K > S₀) If S_T ≤ K: PnL = (S_T − S₀) + Premium ← 保留股票 + 期权金 If S_T > K: PnL = (K − S₀) + Premium ← 股票被行权, 获利封顶 Max Profit = (K − S₀) + Premium ← 有限 Max Loss = S₀ − Premium ← 股价归零 (几乎无限) Breakeven = S₀ − Premium Example (MSFT @ $420): Sell 1 MSFT $440 Call, 30 DTE, Delta 0.30 Premium = $5.60 (mid price) Premium yield = $5.60 / $420 = 1.33%/month If MSFT stays at $420: +$5.60 (+1.33%) ✅ Keep premium If MSFT rises to $460: +$20 + $5.60 − $20 = +$5.60 ✅ But capped If MSFT falls to $390: −$30 + $5.60 = −$24.40 ❌ Cushioned but still down

Operation Steps 操作步骤

1
选股 — 选择你长期看好、愿意持有的股票。优先低波动、稳定上涨的蓝筹 (如MSFT)。
2
买入正股 — 买入100股(1手)。必须持有正股才能卖covered call。
3
选择到期日 — 30-45天到期 (DTE)。太短期权金太少，太长被行权风险大。
4
选择行权价 — Delta ≈ 0.25-0.35 (OTM 5-10%)。Delta越低越安全但期权金越少。
5
卖出Call — Sell to Open 1份Call。确认订单类型为"Sell"而非"Buy"。
6
到期管理 —
• 如果OTM (股价 < 行权价): 期权自动过期，保留期权金，重新卖下一轮
• 如果ITM (股价 > 行权价): 股票被行权卖出。可以重新买入再卖call (见Wheel策略)
• 提前管理: 如果期权亏损>200%, 考虑roll (买回+卖更远期)

2. Cash-Secured Put — 现金担保卖出看跌

Bullish / Neutral Income Assignment Risk

Definition 策略定义

在账户中预留足够现金(= 行权价 × 100)的情况下，卖出1份OTM Put期权。如果股价不跌破行权价，收取全部期权金；如果跌破，被指派以行权价买入100股。适合"想买某只股票但想等更低价格"的投资者。

Payoff Structure 收益结构

Position: Cash = K × 100 + Short 1 Put at strike K (K < S₀) If S_T ≥ K: PnL = +Premium ← Put过期, 保留期权金 If S_T < K: PnL = Premium − (K − S_T) ← 被指派买入, 可能亏损 Max Profit = Premium ← 有限 Max Loss = K − Premium ← 股价归零 Breakeven = K − Premium Example (NVDA @ $140): Sell 1 NVDA $130 Put, 30 DTE, Delta −0.30 Premium = $4.50 (mid) Cash required = $130 × 100 = $13,000 Premium yield = $4.50 / $130 = 3.46%/month If NVDA stays above $130: +$450 (+3.46%) ✅ Free money If NVDA drops to $120: +$450 − $1,000 = −$550 ❌ Assigned at $130, down to $120 Effective buy price if assigned: $130 − $4.50 = $125.50

Operation Steps 操作步骤

1
选股 — 选择你愿意以行权价买入并长期持有的股票。
2
准备现金 — 账户中须有 行权价×100 的现金。否则是"naked put"(保证金要求更高)。
3
选择到期日 — 30-45 DTE。
4
选择行权价 — Delta ≈ −0.25 to −0.35 (OTM 5-10%)。支撑位附近更好。
5
卖出Put — Sell to Open 1份Put。
6
到期管理 —
• OTM: 过期，保留期权金，重复
• ITM: 被指派买入100股。此时转入Covered Call (见Wheel策略)
• 提前止损: 如果亏损 >2× premium, 考虑 Buy to Close

3. The Wheel — 轮转策略

Bullish Income Medium Complexity

Definition 策略定义

Wheel是Covered Call和Cash-Secured Put的组合循环：先卖Put → 被指派后转为Covered Call → Call被行权后回到卖Put。如此循环，持续收取期权金。

The Wheel Cycle: ┌──────────────────────────────────────────────────────────┐ │ │ │ Phase 1: Sell Put │ │ ──────────────── │ │ • 卖OTM Put, 收期权金 │ │ • 如果Put过期 → 回到Phase 1 (循环收钱) │ │ • 如果被指派 → 进入Phase 2 │ │ │ │ Phase 2: Covered Call │ │ ──────────────────── │ │ • 持有100股 + 卖OTM Call │ │ • 如果Call过期 → 回到Phase 2 (继续卖Call) │ │ • 如果Call被行权 → 回到Phase 1 (股票被卖出) │ │ │ └──────────────────────────────────────────────────────────┘ Key: 你始终在收期权金，无论持股还是持现金。

Operation Steps 操作步骤

1
起始 — 选定标的 (推荐MSFT)，准备足够现金买100股。
2
卖Put — Delta −0.30, 30 DTE。
3
如果Put过期 — 收取期权金, 重复Step 2。
4
如果被指派 — 现在持有100股。转到Step 5。
5
卖Call — 行权价 ≥ 你的买入成本 (确保卖出不亏), Delta 0.30, 30 DTE。
6
如果Call过期 — 收取期权金, 重复Step 5。
7
如果Call被行权 — 股票被卖出, 收到现金, 回到Step 2。

Estimated Performance (Based on Backtest)

以MSFT为例: Put phase (SR 0.75) + Call phase (SR 1.00) 交替运行，预计综合年化 10-15%, 胜率 ~75%, 最大风险来自Phase 2期间的股价大跌。

Best Wheel Candidate: MSFT MSFT在两个phase都有正Sharpe，是唯一适合完整Wheel策略的股票。NVDA/TSLA等高波动股虽然期权金更高，但被指派时成本太高，Call phase难以回本。

4. Bull Put Spread — 牛市看跌价差

Bullish Defined Risk Medium Complexity

Definition 策略定义

同时卖出一份高行权价Put + 买入一份低行权价Put (同到期日)。限制了最大亏损的同时收取净期权金。比Naked Put安全得多。

Position: Short 1 Put at K₁ (higher) + Long 1 Put at K₂ (lower) K₂ < K₁ < S₀ If S_T ≥ K₁: PnL = +Net Premium ← 两个Put都过期 If S_T ≤ K₂: PnL = −(K₁ − K₂) + Net Premium ← 最大亏损 (有限!) If K₂ < S_T < K₁: PnL = Premium − (K₁ − S_T) ← 部分亏损 Max Profit = Net Premium ← 有限 Max Loss = (K₁ − K₂) − Net Premium ← 有限! ← 这是关键优势 Width = K₁ − K₂ Example (NVDA @ $140): Sell $130 Put @ $4.50 Buy $120 Put @ $2.00 Net premium = $2.50 Max loss = ($130-$120) − $2.50 = $7.50 Risk/Reward = 7.50 / 2.50 = 3:1 Win rate needed to break even = 75%

Operation Steps 操作步骤

1
选股 — 选择你看涨或看平的股票。
2
选择到期日 — 30-45 DTE。
3
卖高行权价Put — Delta −0.30, 收取较高期权金。
4
买低行权价Put — 低$5-$10, 作为保护。建议同时下单 (spread order)。
5
管理 —
• 盈利 > 50%: 考虑提前Buy to Close锁定利润
• 到期时ITM: 会被指派, 但亏损有限 (最大=spread width − premium)
• 保证金要求: spread width × 100 (远低于CSP的全额现金)

5. Iron Condor — 铁鹰策略

Neutral / Range-bound Income Defined Risk

Definition 策略定义

同时建立一个Bull Put Spread (下方) + Bear Call Spread (上方)。预期股价在一定区间内波动。双向收取期权金，双向定义风险。

Position: Bull Put Spread: Short K₁ Put + Long K₂ Put (K₂ < K₁ < S₀) Bear Call Spread: Short K₃ Call + Long K₄ Call (S₀ < K₃ < K₄) Profit Zone: K₁ < S_T < K₃ ← 股价在两个short strike之间 Max Profit = Total Net Premium ← 两侧premium之和 Max Loss = Wider side width − Total Premium Example (AAPL @ $230): Sell $220 Put / Buy $215 Put → $1.20 credit Sell $240 Call / Buy $245 Call → $1.30 credit Total premium = $2.50 Max loss = $5.00 − $2.50 = $2.50 Win if AAPL stays between $220-$240 (大概率)

Operation Steps 操作步骤

1
选股 — 选择你认为短期不会大涨大跌的股票。低IV环境更好。
2
选择到期日 — 30-45 DTE。
3
下单 — 建议用broker的"Iron Condor"组合下单功能，一次建立4条腿。
4
选择wing width — $5宽度适合大多数$100-$300的股票。
5
管理 —
• 盈利 > 50%: 提前平仓 (Buy to Close整个condor)
• 一侧被突破: 平仓亏损一侧, 保留盈利一侧
• 最大亏损 = wing width − premium (永远已知)

Best Candidates (Based on Our Data)

最适合Iron Condor的股票特征: 低波动 + 窄range + 高IV/RV比 (卖的贵但实际波动小)。根据我们的分析: MSFT (vol 3.8%, 最稳) 和 AAPL (vol 4.9%) 最适合。

B. 入场清单 (每次下单前检查)

  1. ✅ VIX < 25? (VIX > 30时暂停卖期权)
  2. ✅ Earnings在30天之后? (不在财报前卖期权)
  3. ✅ IV Rank > 30%? (当前IV在过去一年中偏高)
  4. ✅ 没有重大宏观事件? (FOMC, CPI, 非农)
  5. ✅ 账户仓位 < 5% 单一标的? (分散风险)
  6. ✅ 已设好止损线? (premium的2x或spread width的50%)

RuleDescription
